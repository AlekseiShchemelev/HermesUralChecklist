# Исправление подключения к Google Script

## Проблема
JSONP запросы падают с ошибкой "Script load error".

## Диагностика

1. Откройте файл `test-script.html` в браузере
2. Нажмите на ссылку "Click to test in new tab"
3. Если видите ошибку 404 или "Script not found" - нужно создать новый deployment

## Создание нового Deployment

### Шаг 1: Откройте Google Apps Script
1. Перейдите на https://script.google.com
2. Найдите ваш проект (или создайте новый)

### Шаг 2: Обновите код
1. Удалите весь старый код
2. Вставьте код из файла `GOOGLE_SCRIPT.js`
3. Сохраните (Ctrl+S)

### Шаг 3: Создайте Deployment
1. Нажмите **Deploy** → **New deployment**
2. Выберите тип: **Web app**
3. Настройки:
   - **Description**: "Checklist API v1"
   - **Execute as**: Me (ваш аккаунт)
   - **Who has access**: Anyone
4. Нажмите **Deploy**
5. Подтвердите разрешения (Allow)

### Шаг 4: Скопируйте URL
1. После деплоя скопируйте **Web app URL**
2. Обновите `js/config.js`:
   ```javascript
   appsScriptUrl: "ВАШ_НОВЫЙ_URL"
   ```

### Шаг 5: Настройте Google Sheet
1. Создайте новую таблицу или откройте существующую
2. В первой строке добавьте заголовки (на русском):
   ```
   ID | ДАТА | СМЕНА | ЦЕХ | ФИО_МАСТЕРА | ПЛАЗМА_ЧЕЛ | ...
   ```
3. В Script Editor нажмите **Run** → **doGet** (для авторизации)
4. Подтвердите разрешения

### Шаг 6: Проверка
1. Откройте в браузере: `ВАШ_URL?action=get`
2. Должны увидеть JSON с данными

## Альтернативное решение (если Google Script не работает)

Если Google Script недоступен, приложение может работать в offline-режиме с localStorage.

Данные сохраняются локально и будут доступны после обновления страницы.
